# MJ Dialog Improvement Plan

## 목표

프로토타입 성격의 앱을 다음 단계로 확장하기 전에, 테스트 가능성, 상태 일관성, 서비스 안정성을 먼저 확보한다.

## 추천 개선 순서

1. 테스트 복구
현재 앱 구조와 맞지 않는 기본 카운터 테스트를 제거하고, 앱이 정상 렌더링되는지 확인하는 스모크 테스트와 핵심 서비스 테스트를 추가한다.

2. 상태 모델 통합
화면이 여러 Riverpod Provider를 직접 조합해 제어하던 구조를 `ChatController` 중심의 단일 세션 상태로 정리한다.

3. 서비스 안정화
STT, TTS, AI 응답 처리 로직에서 초기화 타이밍, 완료 이벤트, 예외 처리, fallback 경로를 명확히 분리한다.

4. 파싱 및 UX 개선
AI 응답은 정규식이 아니라 JSON 파싱 기반으로 처리하고, 사용자에게 보이는 에러 메시지는 기능별로 더 이해하기 쉽게 다듬는다.

5. 문서와 설정 동기화
README, 앱 루트 구성, 디자인/플랫폼 설명을 실제 코드와 맞추고 분석 경고를 함께 줄인다.

## 이번 작업 반영 범위

- `ChatSessionState`와 `ChatController`로 상태/로직 통합
- TTS 완료 기반 상태 전환
- STT 시작/중지 흐름 정리
- Gemma 응답 JSON 파싱 강화
- 앱 루트의 `ProviderScope` 내장
- 현재 구조에 맞는 위젯 테스트 교체
- 정적 분석 경고 정리
